# Adversarial Lightcycle Tournament

Write the strongest bot you can for the supplied simultaneous-action lightcycle arena. Trails become
permanent walls; collisions eliminate players; maps contain obstacles and optional limited boosts.
Use search, territory analysis, opponent modeling, learning, or any other method. You may use public
resources but may not inspect another candidate's workspace or output.

## Competition

Qualification is measured against fixed legal, survival, flood-fill, greedy-territory, and search
bots. The decisive score is a cross-submission round robin over visible and hidden maps, both starting
sides, deterministic seeds, and simultaneous-move edge cases. Report match points, win/draw/loss,
rating with uncertainty, timeouts, and illegal moves.

## Candidate interface

Provide persistent JSON-lines `submission/bot.sh`. It receives game metadata, reset, and turn
records and returns one legal action before the per-turn deadline. It may retain state within a match
but receives no opponent source or private identity.

Provide reproduce.sh and a demo that plays the bot against visible references. Tournament replays
are generated by the trusted evaluator, not selected by candidates. REPORT.md should explain state
representation, territory evaluation, search, simultaneous-action modeling, time management, and
known strategic failures.
